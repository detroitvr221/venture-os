# North Bridge Digital — Heartbeat Automation Checklist

Heartbeat is the recurring automation cycle that keeps the agency responsive and proactive. Every agent participates according to their role.

---

## Heartbeat Schedule

### Every 30 Minutes
- **All agents:** Check Slack for unanswered questions or messages directed at you.
- **Atlas / Relay:** Verify no tasks are orphaned (assigned but unacknowledged).
- **Pulse:** Check workflow health, queue depths, and error rates.

### Every 1 Hour
- **Mercury:** Review pending lead enrichment requests. Run any queued Apollo.io lookups.
- **Scout (VPS1):** Check for pending research requests from other agents.
- **Relay:** Confirm VPS2 agent status and task progress.

### Every 4 Hours
- **Atlas:** Pipeline status review. Compile: active deals, pending proposals, delivery status, blockers.
- **Ledger:** Quick financial pulse — any new charges, subscription alerts, or payment anomalies.
- **Sentinel:** Compliance sweep — any pending reviews, unresolved flags, or expiring consent records.
- **Pulse:** Operational health summary — uptime, incidents, queue trends.

### Every Morning — 8:00 AM ET
- **Atlas:** Generate and post the **Daily Brief** to Slack. Contents:
  - Overnight activity summary
  - Today's priority tasks (ranked)
  - Active client engagements and their status
  - Pipeline snapshot (deals by stage)
  - Any blockers or escalations requiring human attention
  - Agent health status (all agents reporting OK or flagged)
- **Pulse:** System health addendum — overnight incidents, current queue state, any degradation.
- **Ledger:** Financial addendum — yesterday's revenue, outstanding invoices, subscription renewals this week.

---

## Response Contract

At each heartbeat cycle, every agent reports one of:

- **`HEARTBEAT_OK`** — Nothing needs attention. All clear.
- **`HEARTBEAT_ACTION`** — Something needs attention. Include: what, priority level (P1-P4), and recommended action.
- **`HEARTBEAT_BLOCKED`** — Agent is stuck and needs help. Include: what's blocked and what would unblock it.

### Priority Levels

| Level | Meaning | Response Time |
|---|---|---|
| P1 | Critical — revenue impact, system down, compliance violation | Immediate escalation to humans |
| P2 | High — client-facing issue, stuck deal, degraded service | Next heartbeat cycle |
| P3 | Medium — internal inefficiency, upcoming deadline | Next 4-hour review |
| P4 | Low — optimization opportunity, non-urgent improvement | Daily brief |

---

## Heartbeat Failure Protocol

If an agent fails to report for 2 consecutive heartbeat cycles:

1. **Pulse** flags the missing agent in Slack.
2. **Atlas** (or Relay for VPS2) attempts to restart or re-engage the agent.
3. If still unresponsive after 3 cycles, escalate to the human team.
4. Log the failure in Mem0 with timestamp, agent name, and resolution.
